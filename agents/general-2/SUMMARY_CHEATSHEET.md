# Summary Cheatsheet — Kael (general-2)
Quick reference for common summarization tasks.

## TL;DR Crafting
```
1. Read → 2. Extract → 3. Compress → 4. Polish
```

## Summary Types
| Type | When to Use | Output Length |
|------|-------------|---------------|
| **Nano** | Chat responses, quick context | 1-2 sentences |
| **Brief** | Slack/Discord, mobile viewing | 2-4 sentences |
| **Standard** | Email, documentation | 1 paragraph |
| **Detailed** | Reports, stakeholder updates | 2-3 paragraphs |
| **Executive** | C-level, decision-makers | Bullet points + 1 paragraph |

## Key Elements to Preserve
- [ ] **Decision points** — what was decided
- [ ] **Action items** — who does what by when
- [ ] **Blockers** — what's stopping progress
- [ ] **Numbers** — metrics, dates, counts
- [ ] **Risks** — threats or concerns raised

## Compression Techniques
1. **Remove fluff** — "I think", "sort of", "basically"
2. **Combine sentences** — "X happened. Y happened." → "X and Y happened."
3. **Active voice** — "The code was reviewed by Sarah" → "Sarah reviewed the code"
4. **Front-load** — most important info first
